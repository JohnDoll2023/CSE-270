CD

CD path-name

Changes the current directory to path-name, which can be either a word or a list. It’s generally better to use a list so that characters such as slashes in the name are not taken to be logo operators. A tilde (‘~’) can be used as shorthand for your home directory.

CD "~

Pwd

/Users/alan

path-name can either be absolute (starting with a slash) or relative, in which case the directory is changed relative to the current directory.

Pwd

/System

CD [Library]

Pwd

/System/Library

To go up a level, use a double-dot ‘..’.

See also Pwd, Dir, OpenAppend, OpenRead, OpenWrite.

Dir

Dir

List the contents of the current directory. In the resultant list, directories are signified by a trailing slash.

Note that the view of the file system is the real view rather than the logical view seen in the Finder windows. Invisible files (starting with a ‘.’) are listed.

FPrint

FPrint object

(FPrint object1...)

Similar to Print, but writes out to a file rather than to the main command window. Prints out one or more objects to the file currently opened for writing. An object may be a number, word, or list. If an object is a list, the outermost brackets are not printed. Writes a new line afterwards.

FType

FType object

(FType object1...)

Similar to Type, but writes out to a file rather than to the main command window. Writes out one or more objects to the file currently opened for writing. An object may be a number, word, or list. If an object is a list, the outermost brackets are not written. Does not write a new line afterwards.

OpenAppend

OpenAppend file-name

Open file file-name in the current directory for appending, i.e. written data will be added on the end. file-name may be a word or a list. Only one file can be open for writing at a time. After opening, the file can be written to using commands FPrint, FShow, FType. The file should then be closed using CloseWriteFile.

OpenRead

OpenRead file-name

Open file file-name in the current directory for reading. Only one file can be open for reading at a time. After opening, the file can be read from using commands FReadChar, FReadChars, FReadList, FReadWord. The file should then be closed using CloseReadFile.

OpenRead doesn't interpret Unicode files correctly. If you need to read Unicode characters, use OpenTextRead instead.

OpenRead [thing.txt]

Open file file-name in the current directory for reading. Only one file can be open for reading at a time. After opening, the file can be read from using commands FReadChar, FReadChars, FReadList, FReadWord. The file should then be closed using CloseReadFile.

OpenTextRead is able to interpret Unicode files correctly. If you need to read them as a sequence of bytes, use OpenRead instead.

OpenTextWrite

OpenTextWrite file-name

Open file file-name in the current directory for writing to. Previous contents of the file are overwritten. file-name may be a word or a list. Only one file can be open for writing at a time. After opening, the file can be written to using commands FPrint, FShow, FType. The file should then be closed using CloseWriteFile.

OpenTextWrite writes characters in Unicode format - if you want to write the data as ASCII, use OpenWrite.
